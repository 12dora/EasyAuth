# EasyAuth 项目协作规则

## 文档语言

- 本项目所有文档必须使用中文撰写。
- 文档包括但不限于 `README`、`docs/` 下的架构文档、计划文档、ADR、API 文档、试点接入文档和其他说明性 Markdown/RST/TXT 文件。
- 代码标识符、文件路径、命令、协议名、HTTP 路径、API 字段、错误码、配置键、产品名和不可翻译专有名词可以保留英文。
- 新增或修改文档时，必须先检查是否引入了非必要英文正文；如果引入，必须改为中文。

## 项目硬约束

- 项目尚未投入生产。除明确 legacy 白名单、外部系统契约或短期迁移窗口外，不要建立长期兼容层；发现错误模型、口径或流程时，优先一次性修正 schema、domain、API、frontend、tests、docs。
- 遵从第一性原则，不得为罕见、不可能或未定义 case 堆叠投机兜底、静默 fallback 或长期兼容分支；违反业务不变量时应暴露明确错误并修正根因，不得用默认值、吞异常或绕行逻辑隐藏问题。

S12 实现 Authentik OIDC 登录和门户 session 边界

范围

- 新增 Authentik OIDC 登录、callback、授权码交换和 ID token 校验边界。
- 新增门户 session gate，只允许 active UserMirror 访问 `/portal/`。
- 未实现 S13 员工申请表单或 DingTalk 流程。

官方对接依据

- authentik OAuth2/OIDC Provider 授权端点：`/application/o/authorize/`。
- authentik token endpoint：`/application/o/token/`。
- authentik JWKS：`/application/o/<application slug>/jwks/`。
- authentik discovery：`/application/o/<application slug>/.well-known/openid-configuration`。
- OpenID Connect Core：ID token 需要校验 issuer、audience、签名、exp 和 nonce；multi-audience token 需要校验 authorized party (`azp`)。

RED 摘要

- 初始 S12 auth/portal focused tests 在缺少 `/auth/` 和 `/portal/` 路由时失败。
- 复核阶段发现真实 callback 必经 `exchange_authorization_code_for_claims()` stub，未 monkeypatch 时会返回 `400 Invalid OIDC code_exchange: is not implemented`。
- 新增真实 token endpoint + JWKS callback 测试后，要求不 monkeypatch 业务 exchange 函数即可建立 session。

GREEN 实现

- `src/easyauth/accounts/auth.py`：OIDC config、authorization URL、state/nonce、issuer/audience/azp/nonce/sub claims 校验、session 绑定与 session key 轮换。
- `src/easyauth/accounts/oidc_exchange.py`：向 authentik token endpoint 发送 authorization code、client_id、client_secret、redirect_uri；读取 JWKS；验证 RS256 ID token 签名、issuer、audience 和 exp。
- `src/easyauth/accounts/views.py` 和 `src/easyauth/accounts/urls.py`：`/auth/login/`、`/auth/callback/`。
- `src/easyauth/portal/`：`/portal/` session gate。
- `tests/integration/auth/test_oidc_session_s12.py`：登录重定向、真实 fake token/JWKS callback、无效签名、issuer/audience/sub、multi-audience azp、state mismatch、失败清理。
- `tests/integration/portal/test_session_boundary_s12.py`：未登录、active 用户、缺失/disabled 用户 session。

表面 QA

- Django Client 表面脚本通过真实 `exchange_authorization_code_for_claims` 路径，只替换外部 HTTP 层为 fake authentik token/JWKS。
- `/auth/login/` -> 302 到 `https://authentik.example.test/application/o/authorize/`。
- token request 包含配置的 `redirect_uri=http://testserver/auth/callback/`。
- `/auth/callback/?code=...&state=...` -> 302 到 `/portal/`。
- `/portal/` -> 200，响应包含绑定的 Authentik user id。

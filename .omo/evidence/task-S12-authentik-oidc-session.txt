# S12 Authentik OIDC 登录和门户 session 边界证据

## 范围

- 实现 `/auth/login/`，生成 `state` 和 `nonce`，并跳转到 Authentik authorization endpoint。
- 实现 `/auth/callback/`，通过 Authentik token endpoint 换取 `id_token`，用 JWKS 验签并校验 issuer、audience、nonce、subject。
- token exchange 使用配置的 `redirect_uri` 提交给 Authentik，由 Provider 校验 redirect URI。
- OIDC session 绑定 `UserMirror.authentik_user_id`，登录成功后 `cycle_key()` 防止 session fixation。
- 门户 `/portal/` 只允许 active `UserMirror` session 访问；缺失、未知或 disabled session 会清理并重定向登录。
- 未实现 S13 员工申请表单和 DingTalk 审批业务。

## 官方 Authentik 对接依据

- Authorization endpoint：`/application/o/authorize/`
- Token endpoint：`/application/o/token/`
- JWKS：`/application/o/<slug>/jwks/`
- 默认 per-provider issuer：`https://host/application/o/<slug>/`
- Redirect URI 由 Authentik Provider 校验；EasyAuth 在 token exchange 中提交配置的 redirect URI。

## 残余风险

- 当前 OIDC HTTP 实现只支持 HTTPS endpoint 和 RS256/JWKS 验签；如试点 Authentik provider 使用其他 signing algorithm，需要显式配置并补测试。
- `client_secret` 当前来自环境变量；真实部署必须配置 `EASYAUTH_AUTHENTIK_OIDC_CLIENT_SECRET`，否则 callback 会拒绝 code exchange。

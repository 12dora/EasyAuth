S7 静态 app token 和 AppPrincipal 证据记录

范围
- 新增 `AppCredential` 模型和 `applications.0002_appcredential` migration。
- 新增 `AppCredentialService`、`StaticTokenService`、`AppPrincipal`。
- 新增 `AppBearerAuthentication`，解析 Bearer token 并输出 `AppPrincipal`。
- 未实现 S8 权限查询 API，未实现 S9 OAuth2 client credentials。

RED
- 命令：`.tools/uv-venv/bin/uv run --extra dev pytest tests/unit/applications tests/integration/api/test_authentication.py -q`
- 退出码：2
- 关键失败：`ImportError: cannot import name 'AppStaticToken'`、`ImportError: cannot import name 'AppCredential'`、`ModuleNotFoundError: No module named 'easyauth.api.authentication'`。
- 结论：失败原因是 S7 凭据模型、服务和 API authentication 尚未实现，符合 RED 预期。

GREEN
- 新增 token 创建测试：前缀、熵、hash-only 存储、明文不入库。
- 新增 token 认证测试：有效 token、无效 token、disabled token、disabled app、rotate 后新旧 token 短暂并存。
- 新增 DRF authentication 测试：缺失 Authorization 返回 `None`，malformed/invalid token 抛 `AuthenticationFailed`，disabled app 抛 `PermissionDenied`，成功时返回 `AppPrincipal(app_id, app_key, credential_type, credential_id)` 且 `request.auth` 不暴露原始 token。
- 实现使用高熵 `eat_` 前缀 token，数据库只保存 Django `PBKDF2PasswordHasher` hash。token 本身只在创建或轮换结果中返回一次，不写入持久化字段、审计 metadata 或 API authentication 返回值。

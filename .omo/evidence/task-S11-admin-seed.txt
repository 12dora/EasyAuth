S11 强化 Django Admin 和 CRM 种子数据

失败测试:
- `.tools/uv-venv/bin/uv run --extra dev pytest tests/integration/admin/test_applications_admin.py tests/integration/admin/test_seed_crm_pilot.py -q`
- 失败原因：`ModuleNotFoundError: No module named 'easyauth.applications.admin'`。

最小实现:
- 增加 `Role.requestable` 与 `applications.0004_role_requestable` 迁移。
- 在根 URLConf 挂载 `/admin/`。
- 新增 `applications/admin.py`，注册 App、Role、Permission、RolePermission、ApprovalRule、AppCredential。
- AppCredential Admin 不展示 `token_hash`，且禁止通过 Admin 直接新增、修改或删除既有凭据。
- Role Admin 展示 requestable role 的审批规则状态。
- 新增 `seed_crm_pilot` management command，幂等创建 CRM 试点配置、静态 token、seed 用户和当前授权。
